**Bidirectional product sync between Odoo 18 and WooCommerce.**

WoOdoo connects your Odoo ERP with your WooCommerce store, keeping products, categories, images, stock levels, and orders in sync â€” automatically. Configure sync jobs through a modern web UI, get real-time progress via WebSocket, and let scheduled tasks handle the rest.

---

## Features

- **Product sync** â€” Simple and variable products with full variant support (Odoo product.template â†’ WC products/variations)
- **Category sync** â€” Hierarchical categories with topological sorting (parents before children)
- **Image sync** â€” Product images from Odoo uploaded to WordPress Media Library, with SHA-256 deduplication
- **Stock sync** â€” Odoo-authoritative inventory levels pushed to WooCommerce in batches
- **Order sync** â€” WooCommerce orders created as Odoo sale orders with line-item mapping
- **Field-level hashing** â€” SHA-256 delta detection so only changed fields trigger API calls
- **Anti-ping-pong** â€” Sync origin tracking prevents infinite update loops between systems
- **Configurable field mappings** â€” Map any Odoo field to any WC field, per-job, with directional control (Odooâ†’WC, WCâ†’Odoo, bidirectional, skip)
- **Job scheduling** â€” Cron expressions or fixed intervals via Celery Beat
- **Real-time progress** â€” WebSocket endpoint streams sync progress to the frontend
- **Webhook receivers** â€” WooCommerce and Odoo webhooks for near-instant sync triggers
- **Smart matching** â€” Auto-match products by SKUâ†”barcode, or link them manually
- **Review queue** â€” Permanently failed items surface for manual review, retry, or dismissal
- **Connection health monitoring** â€” Periodic health checks with latency tracking and currency validation
- **Setup wizard** â€” Guided first-run setup creates connections + initial sync job
- **Encrypted credentials** â€” Fernet encryption for all stored API keys and passwords
- **Rate-limited auth** â€” JWT authentication with bcrypt password hashing and login rate limiting

## Quick Start

### Prerequisites

- Docker and Docker Compose v2
- Python 3.12+ (only for key generation)

### 1. Clone and configure

```bash
git clone git@github.com:Abdulee/WoOdoo.git
cd WoOdoo

# Copy the example environment file
cp .env.example .env

# Generate secure keys (SECRET_KEY + FERNET_KEY)
bash scripts/generate-keys.sh --write
```

### 2. Edit `.env`

At minimum, change these values:

```env
ADMIN_PASSWORD=your-secure-password
SECRET_KEY=<auto-generated>
FERNET_KEY=<auto-generated>
```

### 3. Start all services

```bash
docker compose up -d
```

This starts 6 containers:

| Service | Container | Port |
|---------|-----------|------|
| PostgreSQL 16 | woodoo-postgres | 5432 |
| Redis 7 | woodoo-redis | 6379 |
| FastAPI backend | woodoo-app | 8000 |
| Celery worker | woodoo-worker | â€” |
| Celery Beat | woodoo-beat | â€” |
| Next.js frontend | woodoo-frontend | 3000 |

### 4. Open the UI

Navigate to [http://localhost:3000](http://localhost:3000). The setup wizard will guide you through:

1. Creating an **Odoo connection** (URL, database, username, API key)
2. Creating a **WooCommerce connection** (store URL, consumer key/secret)
3. Testing both connections
4. Creating your first sync job

### 5. Verify

```bash
# Check backend health
curl http://localhost:8000/api/health

# Check all services are running
docker compose ps
```

## Usage Overview

If this is your first time using WoOdoo, the fastest working path is:

1. Start services with Docker Compose
2. Log in at `http://localhost:3000`
3. Complete Setup Wizard (`/setup`)
4. Create your first sync job
5. Trigger a manual run from **Sync Jobs**
6. Monitor progress from Dashboard + Sync Logs

### Connections

Add your Odoo and WooCommerce credentials. WoOdoo encrypts all secrets at rest with Fernet and tests connectivity (including WordPress Media API access for image sync).

### Sync Jobs

Create jobs that define:
- **Direction** â€” Odooâ†’WC, WCâ†’Odoo, or bidirectional
- **Field mappings** â€” Which Odoo fields map to which WC fields, and in which direction
- **Filters** â€” Only sync products matching specific criteria
- **Schedule** â€” Cron (`0 */6 * * *`) or interval (every N seconds)
- **Lifecycle rules** â€” What to do when a source product is new or deleted

### Monitoring

- **Dashboard** â€” Overview of recent executions, success/error counts
- **Logs** â€” Detailed per-execution sync logs with filtering
- **Real-time** â€” WebSocket-powered live progress during sync runs
- **Review Queue** â€” Surface permanently failed items for manual action

### Product Matching

For existing catalogs that overlap, use auto-match (SKUâ†”barcode) or manual linking through the Explorer page.
