# WoOdoo

**Bidirectional product sync between Odoo 18 and WooCommerce.**

WoOdoo connects your Odoo ERP with your WooCommerce store, keeping products, categories, images, stock levels, and orders in sync — automatically. Configure sync jobs through a modern web UI, get real-time progress via WebSocket, and let scheduled tasks handle the rest.

---

## Features

- **Product sync** — Simple and variable products with full variant support (Odoo product.template → WC products/variations)
- **Category sync** — Hierarchical categories with topological sorting (parents before children)
- **Image sync** — Product images from Odoo uploaded to WordPress Media Library, with SHA-256 deduplication
- **Stock sync** — Odoo-authoritative inventory levels pushed to WooCommerce in batches
- **Order sync** — WooCommerce orders created as Odoo sale orders with line-item mapping
- **Field-level hashing** — SHA-256 delta detection so only changed fields trigger API calls
- **Anti-ping-pong** — Sync origin tracking prevents infinite update loops between systems
- **Configurable field mappings** — Map any Odoo field to any WC field, per-job, with directional control (Odoo→WC, WC→Odoo, bidirectional, skip)
- **Job scheduling** — Cron expressions or fixed intervals via Celery Beat
- **Real-time progress** — WebSocket endpoint streams sync progress to the frontend
- **Webhook receivers** — WooCommerce and Odoo webhooks for near-instant sync triggers
- **Smart matching** — Auto-match products by SKU↔barcode, or link them manually
- **Review queue** — Permanently failed items surface for manual review, retry, or dismissal
- **Connection health monitoring** — Periodic health checks with latency tracking and currency validation
- **Setup wizard** — Guided first-run setup creates connections + initial sync job
- **Encrypted credentials** — Fernet encryption for all stored API keys and passwords
- **Rate-limited auth** — JWT authentication with bcrypt password hashing and login rate limiting

## Quick Start

### Prerequisites

- Docker and Docker Compose v2
- Python 3.12+ (only for key generation)

### 1. Clone and configure

```bash
git clone https://github.com/your-org/woodoo.git
cd woodoo

# Copy the example environment file
cp .env.example .env

# Generate secure keys (SECRET_KEY + FERNET_KEY)
bash scripts/generate-keys.sh --write
```

### 2. Edit `.env`

At minimum, change these values:

```env
ADMIN_PASSWORD=your-secure-password
SECRET_KEY=<auto-generated>
FERNET_KEY=<auto-generated>
```

### 3. Start all services

```bash
docker compose up -d
```

This starts 6 containers:

| Service | Container | Port |
|---------|-----------|------|
| PostgreSQL 16 | woodoo-postgres | 5432 |
| Redis 7 | woodoo-redis | 6379 |
| FastAPI backend | woodoo-app | 8000 |
| Celery worker | woodoo-worker | — |
| Celery Beat | woodoo-beat | — |
| Next.js frontend | woodoo-frontend | 3000 |

### 4. Open the UI

Navigate to [http://localhost:3000](http://localhost:3000). The setup wizard will guide you through:

1. Creating an **Odoo connection** (URL, database, username, API key)
2. Creating a **WooCommerce connection** (store URL, consumer key/secret)
3. Testing both connections
4. Creating your first sync job

### 5. Verify

```bash
# Check backend health
curl http://localhost:8000/api/health

# Check all services are running
docker compose ps
```

## Usage Overview

### Connections

Add your Odoo and WooCommerce credentials. WoOdoo encrypts all secrets at rest with Fernet and tests connectivity (including WordPress Media API access for image sync).

### Sync Jobs

Create jobs that define:
- **Direction** — Odoo→WC, WC→Odoo, or bidirectional
- **Field mappings** — Which Odoo fields map to which WC fields, and in which direction
- **Filters** — Only sync products matching specific criteria
- **Schedule** — Cron (`0 */6 * * *`) or interval (every N seconds)
- **Lifecycle rules** — What to do when a source product is new or deleted

### Monitoring

- **Dashboard** — Overview of recent executions, success/error counts
- **Logs** — Detailed per-execution sync logs with filtering
- **Real-time** — WebSocket-powered live progress during sync runs
- **Review Queue** — Surface permanently failed items for manual action

### Product Matching

For existing catalogs that overlap, use auto-match (SKU↔barcode) or manual linking through the Explorer page.
